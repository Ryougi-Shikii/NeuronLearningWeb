Memory

LSTMs solve the vanishing gradient problem by introducing a dedicated memory cell and gating mechanisms that control what to remember, forget, and output.

The Problem LSTM Solves

Vanilla RNNs fail to learn long-range dependencies because gradients vanish over many time steps. The network effectively forgets what happened more than ~10 steps ago.

LSTM (Hochreiter & Schmidhuber, 1997) introduces a cell state — a separate "memory highway" that can carry information across hundreds of steps with minimal modification, solving the gradient problem.

Analogy Imagine reading a 500-page novel. A vanilla RNN is like only remembering the last paragraph. An LSTM is like keeping sticky notes — you decide what's important to write down (input gate), what old notes to throw away (forget gate), and what to read when you need to answer a question (output gate).

The Cell State — The Memory Highway

The cell state Cₜ runs through the entire sequence, only modified by two gentle operations — a multiplicative forget and an additive input. This additive path is what allows gradients to flow without vanishing.

Cₜ = fₜ ⊙ C₍ₜ₋₁₎ + iₜ ⊙ C̃ₜ ↑ ↑ (forget old) (add new info)

The ⊙ symbol means element-wise multiplication. fₜ decides what fraction of the old cell state to keep. iₜ ⊙ C̃ₜ is the new candidate information weighted by how much to add.

LSTM cell — the orange highway is the cell state, gates modulate what passes through

The Three Gates

Every gate is a sigmoid-activated layer outputting values in (0,1). Think of 0 as "block everything" and 1 as "let everything through".

Forget Gate fₜ

Decides what to erase from the cell state. fₜ = σ(Wf · [h₍ₜ₋₁₎, xₜ] + bf). Near 0 → forget. Near 1 → keep.

Input Gate iₜ

Decides what new information to write. iₜ = σ(Wi · [h, x] + bi). Controls how much of the candidate C̃ₜ to add.

Output Gate oₜ

Decides what to expose as hidden state. hₜ = oₜ ⊙ tanh(Cₜ). The output that flows to the next layer.

Why sigmoid gates work Sigmoid squashes to (0,1). Element-wise multiplying by a sigmoid gate is like a soft switch — you can partially remember, partially forget. This differentiability is what makes the gradients flow during training.

GRU — Gated Recurrent Unit

GRU (Cho et al., 2014) is a simplified LSTM. It merges the cell state and hidden state, and uses just two gates instead of three — making it faster to train with similar performance on many tasks.

Reset Gate

Controls how much of the previous hidden state to forget when computing the candidate.

Update Gate

Combines the forget and input gate roles — decides how much of the new candidate to blend with the old state.

LSTM vs GRU LSTM has more parameters and more expressive power. GRU is faster and works better on smaller datasets. In practice, try GRU first — it often matches LSTM with less compute. Both are largely superseded by Transformers for long sequences.
