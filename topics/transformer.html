The Transformer

"Attention is All You Need" (Vaswani et al., 2017) introduced an architecture that replaced recurrence entirely with attention mechanisms — enabling massive parallelism and powering GPT, BERT, and every modern LLM.

Why Replace RNNs?

RNNs and LSTMs are sequential — step 5 depends on step 4, which depends on step 3... You can't parallelise this. For a 1,000-word document, that's 1,000 sequential operations.

Transformers process the entire sequence at once using attention — every position can look at every other position in a single step. This parallelism made it feasible to train on internet-scale data.

The key trade-off RNNs: O(n) sequential steps, O(1) memory per step. Transformers: O(1) sequential steps, O(n²) attention computation. The n² cost made large Transformers expensive — but GPUs handle parallelism far better than sequences, so Transformers win in practice.

Self-Attention — How It Works

For each token in the sequence, self-attention computes a weighted sum of all other tokens' values. The weights (attention scores) determine how much each token should "attend to" each other token.

Each token produces three vectors from its embedding:

Query (Q)

"What am I looking for?" — what this position wants to find in the context.

Key (K)

"What do I contain?" — what each position offers to other queries.

Value (V)

"What do I contribute?" — the actual content to be aggregated if attended to.

Attention(Q, K, V) = softmax( QKᵀ / √dₖ ) · V QKᵀ → similarity scores between all query-key pairs / √dₖ → scale to prevent very small gradients (dₖ = key dimension) softmax → turn scores into probabilities (sum to 1) · V → weighted sum of values
Self-attention: "cat" attends to all tokens, weighted by relevance. Arrow thickness = attention weight.

Multi-Head Attention

A single attention head learns one type of relationship. Multi-head attention runs h attention operations in parallel, each with its own Q, K, V projection — then concatenates the results.

MultiHead(Q,K,V) = Concat(head₁, head₂, ..., headₕ) · Wᴼ headᵢ = Attention(Q·Wᵢᴼ, K·Wᵢᴷ, V·Wᵢᵛ)
Head 1

Might attend to syntactic relationships — subject-verb agreement.

Head 2

Might attend to semantic similarity — words that mean similar things.

Head 3

Might capture positional patterns — previous/next token relationships.

Together

Each head specialises automatically during training. The concatenation captures richer representations.

Positional Encoding

Attention is permutation invariant — "cat sat on mat" and "mat on sat cat" would produce identical attention weights without positional information. We fix this by adding positional encodings to the input embeddings.

PE(pos, 2i) = sin(pos / 10000^(2i/dmodel)) PE(pos, 2i+1) = cos(pos / 10000^(2i/dmodel))

These sinusoidal functions have a key property: PE(pos+k) can be represented as a linear function of PE(pos), so the model can learn to attend by relative position.

Modern alternative Learned positional embeddings (used in BERT, GPT) work equally well. Rotary Position Embedding (RoPE, used in LLaMA) encodes relative positions directly into the Q and K vectors — enabling better length generalisation.

The Full Architecture

The original Transformer has an encoder (understands input) and a decoder (generates output). Models like BERT use only the encoder; GPT uses only the decoder.

ENCODER BLOCK (×N): x → LayerNorm → MultiHead Self-Attention → residual + → LayerNorm → Feed-Forward Network → residual + DECODER BLOCK (×N): y → LayerNorm → Masked Self-Attention → residual + → LayerNorm → Cross-Attention (attends to encoder) → residual + → LayerNorm → Feed-Forward Network → residual +
Residual Connections

Skip connections (from ResNet) add the input back to the output of each sublayer — preventing vanishing gradients in deep stacks.

Layer Norm

Normalises activations within each layer — stabilises training of very deep Transformers.

Feed-Forward

Two linear layers with ReLU in between, applied independently per position. Expands and contracts dimension: dmodel → 4·dmodel → dmodel.

BERT / GPT

BERT = encoder-only, bidirectional — great for classification. GPT = decoder-only, left-to-right — great for generation.

Why Transformers Dominate

They parallelise perfectly on GPUs, scale predictably with data and parameters (scaling laws), and attention can model any kind of relationship regardless of distance. Every state-of-the-art model in NLP, vision (ViT), audio (Whisper), and protein folding (AlphaFold) is a Transformer.
