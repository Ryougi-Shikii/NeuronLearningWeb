Convolutional
Neural Networks

CNNs revolutionized computer vision by learning spatial hierarchies of features directly from images — no hand-crafted feature engineering needed.

Why Not Just Use ANN?

A 224×224 RGB image has 150,528 inputs. Connecting those fully to even a modest hidden layer means millions of parameters — expensive, and it ignores spatial structure. A pixel's neighbours matter; far-away pixels usually don't.

CNNs exploit translation invariance: a cat in the top-left and a cat in the bottom-right should activate the same detector. They do this with shared weights via convolution.

Key insight Instead of one weight per connection, CNNs use a small filter (kernel) that slides across the entire image — the same weights used everywhere. This massively reduces parameters while preserving spatial relationships.

The Convolution Operation

A filter (kernel) is a small matrix of learnable weights (e.g. 3×3). It slides across the input image computing a dot product at each position, producing a feature map.

Feature map(i,j) = Σ Σ Input(i+m, j+n) · Filter(m,n) m n
Filter / Kernel

Small learnable weight matrix. Different filters detect different features: edges, curves, textures.

Feature Map

Output of applying one filter across the full input. Highlights where that filter's pattern appears.

Stride

Step size of the sliding filter. Stride=2 halves the output dimensions.

Padding

'Same' padding adds zeros around the border so output matches input size. 'Valid' = no padding.

A 3×3 filter slides over a 5×5 input producing a 3×3 feature map (stride=1, no padding)

Pooling Layers

Pooling reduces the spatial dimensions of feature maps, making the network more computationally efficient and providing some translation invariance.

Max Pooling

Takes the maximum value in each window (e.g. 2×2). Preserves the strongest activations. Most common.

Average Pooling

Takes the average. Preserves overall information better, used in some architectures (e.g. GoogLeNet).

Global Average Pooling

Collapses each entire feature map to a single number. Used just before the output layer in modern CNNs.

Why max pooling works If a feature detector fires somewhere in a region, we care that it fired — not exactly where. Max pooling captures the "was it there?" question, giving robustness to small translations.

Full CNN Pipeline

A typical CNN stacks these operations in order:

Input Image → [Conv → ReLU] × N (feature extraction) → MaxPool (downsampling) → [Conv → ReLU] × N (deeper features) → MaxPool → Flatten → Fully Connected Layers (classification) → Softmax Output

Famous Architectures

LeNet-5 (1998)

First practical CNN by LeCun. Digit recognition on MNIST. Proved CNNs could learn from images end-to-end.

AlexNet (2012)

Won ImageNet by a huge margin. Introduced ReLU, dropout, and GPU training. Sparked the deep learning revolution.

VGGNet (2014)

Used only 3×3 filters throughout. Showed depth matters. Simple, elegant, still used as a backbone.

ResNet (2015)

Introduced skip connections (residual blocks). Enabled training of 100+ layer networks by solving vanishing gradients.
