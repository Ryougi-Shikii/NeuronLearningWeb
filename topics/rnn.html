Recurrent Neural
Networks

RNNs process sequences by maintaining a hidden state — a memory of what came before. They're the first architecture designed for time-series, text, and audio.

Why Sequences Need Different Networks

Standard ANNs assume inputs are independent. But language, audio, and time-series data have order that matters. "Dog bites man" ≠ "Man bites dog".

RNNs solve this by processing one element at a time while passing a hidden state forward — a compressed summary of everything seen so far.

Analogy Reading a sentence word by word. At each word, you update your understanding. The hidden state is your running mental model of what the text means.
Unrolled RNN — same weights used at every time step, hidden state h passes forward

The Hidden State Equation

At each time step t, the RNN takes the current input xₜ and the previous hidden state h₍ₜ₋₁₎, combines them, and produces the new hidden state hₜ.

hₜ = tanh( Wₕ · h₍ₜ₋₁₎ + Wₓ · xₜ + b ) yₜ = Wᵧ · hₜ + bᵧ
Wₕ

Recurrent weight matrix — how much the previous hidden state influences the current one.

Wₓ

Input weight matrix — how much the current input contributes.

Weight sharing

The same Wₕ, Wₓ are used at every time step — the network learns one set of rules that applies everywhere in the sequence.

Backpropagation Through Time (BPTT)

Training an RNN means unrolling it across time steps and applying the chain rule backwards — from the final output all the way back to the first time step.

∂L/∂Wₕ = Σₜ ∂Lₜ/∂Wₕ Each ∂Lₜ/∂Wₕ requires multiplying gradients back through all earlier time steps.
Why this matters Gradients are multiplied at every time step. If the weight matrix has eigenvalues <1, gradients shrink exponentially (vanishing). If >1, they explode. Long sequences make this catastrophic.

Limitations of Vanilla RNN

Vanishing Gradients

Gradients shrink to zero over long sequences — the network forgets early inputs. The "long-range dependency" problem.

Exploding Gradients

Gradients grow exponentially. Fixed by gradient clipping, but it's a hack, not a solution.

Sequential

Processing is strictly left-to-right — can't be parallelised. Slow to train on long sequences.

Short Memory

In practice, vanilla RNNs struggle to connect information more than ~10 steps apart.

These limitations motivated the invention of LSTM — the next topic.
